前端打点项目

这个项目是所有前端性能打点的集合地，以后前端打点的功能请在此处修改，请保持真爱

文件目录介绍

  1. core : 核心js文件
  2. base : 基础(全局)的一些统计js文件
  3. modules : 模块的拓展功能js文件

modules文件目录介绍

以globe文件夹为主

  1. log.js 子模块自己要添加的一些统计模块请写在此处(具体模块写法请参考下面的代码示例)
  2. config.js 用于grunt打包生成(grunt打包请参考下面的模块打包)
  3. dist 打包生成的js，没有太多用处，可以让你看下生成的文件是不是你想要的= =

统计模块写法规范

参照 base/error.js

init方法可有可无，如果有init方法注入的时候会执行init函数

exports 方法必须有 而且必须返回一个{key : value}这样的obj, 否则不会让其变成模块

最后注入LogApp LogApp.extend('test',PageError); 注意注入后的名字是test喔！

Grunt打包注意事项(grunt请先安装! git clone下来之后请执行一把npm install即可)

打包代码示例：grunt setlog:modules/globe/config.json(config.js请自行配置,谢谢各位大爷！)

基础模块打包执行

grunt

生成的文件为

dist/md-log.js

自定义模块不用合并，直接调用基础模块注入的全局对象
